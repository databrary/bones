dashboard

Dashboard overview

–>

Purpose

This is a user’s dashboard or primary landing page. It is where users land after they log-in from the Databrary home page.

Routes

API

  • databrary.org/my
  • databrary.org/dashboard

From

  • login
  • databrary.org

To

Header links:

  • search
  • news
  • about
    • aboutDatabrary
    • aboutProject consider calling this something other than “project”?
    • aboutTeam
  • community
    • people
    • orgs
  • support
  • avatar
    • dashboard-auth (dashboard-noAuth)
    • settings
      • profile
      • accounts
      • email
      • security
    • myProjects
    • logout
  • login
  • register

Direct links:

  • thisProject from “Create project” button
  • manageProjects page through “Edit project permissions” button
  • auditProjects page through “Audit my projects” button
  • settings through “Update my profile” button
  • search page through “search people” button
  • search results page of all users belonging to /my/org through “browse colleagues” button
  • [person] public pages of other users page from the “people” section
  • [org] public page of user’s own institution page from the “people” section
  • [static pages] mostly from the “support” section of the header links

Actions

  • Create project (modal - takes user to thisProject on save)
  • View projects
    • recently updated (default)
    • shared by others (option)
    • belonging to specific user groups (option)
    • belonging to specific project groups (option)
  • Search for projects
  • Update profile button takes user to settings
  • View people
    • collaborators (default)
    • colleagues (default)
  • Search for people
  • View latest (public + shared) projects on Databrary
  • ~Manage people~
  • ~Edit profile~ (always available through header avatar)
  • ~Edit other settings~ (always available through header avatar)
  • ~Learn about Databrary~ (always available through header avatar)
